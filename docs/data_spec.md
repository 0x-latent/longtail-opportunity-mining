# 数据交付规范

本文档说明原始数据和标签表的格式要求，请标注方按此规范交付。

---

## 文件命名规范

每个项目交付两个文件，命名格式：

```
{项目名}_{数据类型}_{日期}.csv
```

| 文件 | 命名格式 | 示例 |
|------|---------|------|
| 原始数据 | `{项目名}_posts_{YYYYMMDD}.csv` | `cold_cough_posts_20260401.csv` |
| 标签表 | `{项目名}_labels_{YYYYMMDD}.csv` | `cold_cough_labels_20260401.csv` |

**规则**：
- 项目名使用英文小写+下划线（如 `cold_cough`、`baby_skin`、`allergy`）
- 日期为交付日期
- 同一项目的两个文件使用相同的项目名和日期
- 如有数据更新，日期递增即可（如 `cold_cough_labels_20260415.csv`）

---

## 一、原始数据（{项目名}_posts_{日期}.csv）

### 格式要求

- 文件格式：CSV，UTF-8 编码（带 BOM 亦可）
- 每行一条帖子
- `text` 为唯一必填字段，其余可选
- 保留帖子原文，不要预处理（不要去除表情、话题标签等，pipeline 会自动清洗）

### 字段说明

| 字段名 | 必填 | 类型 | 说明 |
|--------|------|------|------|
| id | 否 | 文本 | 帖子唯一标识，留空则由系统自动生成 |
| platform | 否 | 文本 | 来源平台（如：小红书、抖音、微博、宝宝树） |
| date | 否 | 文本 | 发布日期，格式 yyyy-MM-dd（如 2026-03-15），未知可留空 |
| title | 否 | 文本 | 帖子标题，没有标题的平台可留空 |
| text | **是** | 文本 | 帖子正文，保留原始内容 |

### 示例

```csv
id,platform,date,title,text
,小红书,2026-01-10,宝宝反复咳嗽怎么办,三岁宝宝反复咳嗽有痰快一个月了，吃了头孢和氨溴索都不见好，有没有宝妈有经验的 #宝妈日常 @DOU+小助手
,抖音,2026-02-03,,晚上咳嗽咳到睡不着，喝了蜂蜜水稍微好一点，但是第二天又开始了，崩溃
,微博,2026-01-28,支原体肺炎住院记录,孩子支原体肺炎住院第五天，从阿奇换成了多西环素，今天终于不烧了，分享下经历给同样焦虑的家长
,宝宝树,,,四个月宝宝拉肚子拉了三天，吃了蒙脱石散和益生菌，请问还需要去医院吗
,小红书,2026-03-01,一岁半湿疹反反复复,宝宝湿疹从出生到现在反反复复，试了好多药膏都不行，最近在吃益生菌调理，希望能好起来
```

### 注意事项

1. 文本中的 URL、话题标签（#xxx#）、@提及、表情符号等**不需要手动清除**，系统会自动处理
2. 如果原始数据列名不是上述英文名（如"文本"、"平台"），请在交付时附注列名对照关系
3. 如有多个 CSV 文件，保持列名一致即可，系统会自动合并处理

## 二、标签表（{项目名}_labels_{日期}.csv）

### 格式要求

- 文件格式：CSV，UTF-8 编码
- 每行一个标签条目
- 两列：`label_path`（标签路径）和 `keywords`（关键词列表）

### 字段说明

| 字段名 | 必填 | 说明 |
|--------|------|------|
| label_path | **是** | 标签层级路径，用 `/` 分隔，层数不限。第一层为维度名（如"症状"、"场景"） |
| keywords | **是** | 该标签对应的**原文关键词**，用英文分号 `;` 分隔。必须是帖子中实际会出现的词语 |

### 层级路径规则

```
第一层/第二层/第三层/.../最后一层
维度    分类    子分类     具体标签
```

- **第一层**（维度）：系统会自动为每个维度生成掩码标记（如 `[症状]`、`[场景]`），用于文本掩码
- **中间层**（分类）：仅用于组织归类，层数自由，可以是 0 层也可以是多层
- **最后一层**（标签）：最具体的标签名称

### 示例

```csv
label_path,keywords
症状/咽喉/咳嗽,咳嗽;一直咳;干咳;痰咳;咳个不停;咳得厉害
症状/咽喉/喉咙痛,喉咙痛;嗓子痛;咽痛;喉咙不舒服;嗓子发炎;嗓子疼
症状/鼻部/流鼻涕,流鼻涕;鼻涕;鼻塞;鼻子不通气;鼻涕很多;擤鼻涕
症状/全身/发烧,发烧;发热;低烧;高烧;体温高;烧到39度;反复发烧
场景/时间/夜间,晚上;夜里;半夜;睡前;夜间;凌晨
场景/人物/幼儿,小孩;孩子;宝宝;幼儿;娃;崽崽;小朋友
场景/地点/幼儿园,幼儿园;上学;入园;园里;班上
方案/西药/美林,美林;布洛芬;布洛芬混悬液
方案/西药/头孢,头孢;头孢克肟;头孢克洛
方案/中医/推拿,小儿推拿;推拿;按摩;捏脊
方案/食疗/蜂蜜水,蜂蜜水;蜂蜜柠檬;蜂蜜
效果评价/正面/有效,有用;管用;见效快;效果好;好多了;终于好了
效果评价/负面/无效,没用;不管用;没效果;白吃了;一点用没有
情绪/焦虑,焦虑;担心;害怕;慌;急死了;愁死了
情绪/崩溃,崩溃;难受;顶不住;受不了;太难了
```

### 关键词填写要求

1. **必须是原文中实际出现的词语**，不是标签的规范名称。例如标签是"发烧"，关键词应包含用户常说的"烧到39度"、"反复发烧"等表述
2. **同义词、口语化表达都要列出**。例如"喉咙痛"还应包含"嗓子疼"、"咽痛"等
3. **长短搭配**。既要有短词（"咳嗽"）也要有短语（"咳个不停"、"一直咳"）
4. 关键词之间用**英文分号** `;` 分隔，不要用中文分号 `；`
5. 关键词**不需要穷举**，覆盖最常见的表述即可（建议每个标签 3-10 个关键词）
6. 如果一个关键词可能属于多个标签，放在**最相关**的那个标签下

### 层级设计建议

- 第一层维度不宜过多，建议 4-8 个（如：症状、场景、方案、效果评价、情绪、人群）
- 层级深度根据实际需要，2-4 层较为常见
- 同一维度下的标签**关键词不要重叠**（如"咳嗽"不要同时出现在"干咳"和"痰咳"的关键词中）
